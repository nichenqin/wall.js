Why wall.js

Built for real pages, not demos only.

State, not array tricks

Fixed DOM order + index state. No reverse queues, no z-index roulette. Easy to debug.

Modern input

wheel, Pointer Events, and e.key. Nested scroll works — page change only at edges.

Destroyable

AbortController cleans listeners. Styles restore on destroy(). SPA-safe.

Motion-aware

Web Animations API with CSS fallback. Honors prefers-reduced-motion.

API

Small surface. Enough power.

Drop in a wrapper, mark slides with data attributes, listen for change. Full reference in docs/API.md.

  • next() / prev() / goTo(i)
  • nextSlide() / prevSlide()
  • on('change', handler) — compose hash / analytics
  • destroy()
import { Wall } from 'wall.js'

const wall = new Wall('#wall')
const hashes = ['top', 'features', 'api', 'slides', 'install']

// App layer: bind page → URL (not core)
wall.on('change', ({ to }) => {
  history.replaceState(null, '', `#${hashes[to]}`)
})

// Deep link: /#api → goTo section
const i = hashes.indexOf(location.hash.slice(1))
if (i >= 0) wall.goTo(i)

Install ↓

Get started

Install in one line.

pnpm add wall.js
pnpm add wall.js @wall.js/react
pnpm add wall.js @wall.js/vue
pnpm add wall.js @wall.js/svelte
// React
import { Wall } from '@wall.js/react'
<Wall><Wall.Section /></Wall>

// Vue
import { Wall, WallSection } from '@wall.js/vue'
<Wall><WallSection /></Wall>

// Svelte
import { Wall, WallSection } from '@wall.js/svelte'
<Wall><WallSection /></Wall>
